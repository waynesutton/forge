# Forge

Self-hostable Discord form builder and approval engine.

Build forms in a web dashboard, submit them from Discord via slash commands, route submissions through an optional mod queue, and publish approved answers into any text or forum channel. Real time end to end via Convex.

Forge is a developer tool for teams that already live inside Discord. Design a form in the browser, publish it as a slash command with one click, and keep every submission, decision, and ticket action in a searchable audit log. It ships as a single Convex deployment, so there is no extra infrastructure to run, no worker process to babysit, and no webhook plumbing to maintain. Good for applications, bug bounties, support tickets, internal requests, and anything else you would otherwise wire up with three bots and a spreadsheet.

Built for full-stack developers and platform teams. Type-safe from Convex queries to React UI, reactive subscriptions instead of polling, Ed25519 signature verification on every Discord interaction, and a static frontend hosted on Convex alongside the backend.

## What it does

- Design forms in a visual editor with a live Discord modal preview
- Register each form as a guild slash command in one click
- Collect submissions through Discord native modals, no leaving the app
- Route to a mod queue with Approve and Deny buttons, or publish straight to a channel
- Publish into text channels or forum threads with title templates and tags
- Run tickets with lifecycle buttons (claim, resolve, close, reopen) and auto close timers
- Review submissions in a results page with CSV and PDF export, hide, delete, and reply in Discord
- Read a per form audit log with plain English hints for Discord REST errors

## Features

- Field types: short text, paragraph, email (private by default), code block with copy, single select, yes or no, checkbox, and number with min, max, and currency unit
- Required, blocked, and moderator role gates read from the live guild role cache
- Per user and per day submission caps with custom success messages
- Private fields stripped from public channels but kept in mod queue and dashboard views
- Optional submitter link on publish, no pings
- Moderator name in embed footer toggle
- Bot authored reply composer that posts back into the published thread or channel
- Opt in Discord message cleanup when a submission is deleted in the dashboard
- Hourly cron sweep that auto closes inactive tickets and archives forum threads

## Stack

- React 19, Vite 6, Tailwind v4, React Router 7
- Convex backend with queries, mutations, actions, and crons
- Convex Auth via `@robelest/convex-auth` with GitHub OAuth
- Static hosting via `@convex-dev/static-hosting`
- Discord Interactions API with Ed25519 verification
- `@dnd-kit` for the field reorder, Phosphor icons, `jspdf` for PDF export

## Getting started

Prereqs: Node 20+, a Convex account, a Discord application, and a GitHub OAuth app.

```bash
npm install
cp .env.example .env.local
npx convex dev
npm run dev
```

Fill in `.env.local` and set the same secrets in the Convex deployment:

```bash
npx convex env set DISCORD_PUBLIC_KEY ...
npx convex env set DISCORD_APPLICATION_ID ...
npx convex env set DISCORD_BOT_TOKEN ...
npx convex env set DISCORD_CLIENT_SECRET ...
npx convex env set AUTH_GITHUB_ID ...
npx convex env set AUTH_GITHUB_SECRET ...
```

`JWT_PRIVATE_KEY` and `JWKS` are generated by the Robel auth CLI. See `docs/discord-setup.md` for the full Discord app walkthrough.

## Access model

Sign in with GitHub. Access is gated to an allow list with one deterministic owner and the rest as admins. Configure the rules in `convex/lib/access.ts`.
